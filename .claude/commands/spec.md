Load and review a specific section of the project before working on it.

Usage: /spec <topic>

Topic: $ARGUMENTS

Steps:
1. Read `CLAUDE.md` for project rules and current state.
2. Based on the topic, read the relevant documentation:
   - Architecture → `docs/architecture.md`
   - Experiments → `docs/experiments.md` + `LEARNINGS.md`
   - Problem/thesis → `docs/problem.md`
   - Future work → `docs/future-work.md`
   - Development process → `docs/development_process.md`
   - Research landscape → `docs/research_landscape.md`
   - Reading list → `readinglist.md`
3. Read any relevant existing code (check both `main` and `v1` branches if needed).
4. Summarize:
   - What exists for this topic
   - What's been tried and what the results were
   - What the current state is
   - What decisions are still open
